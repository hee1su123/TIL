# UDP 프로토콜
- 비연결형, 비신뢰성 전송 프로토콜
- 3 way handshaking 과 같은 연결 수립 과정이 없음
- TCP 에 비해 오류 검사 과정이 체크섬 외에는 없음
- header 가 비교적으로 작음
- PDU(protocol data unit) : datagram

## 장점
- 빠른 속도
    - 적은 양의 오버헤드(특정 기능을 수행하는데 필요한 시간, 메모리, 자원)
    - 작은 헤더
    - 연결을 수립, 유지 하는 비용이 없음
    - 훼손, 손실된 데이터를 감지 하지 않음

## 단점
- 데이터의 신뢰성, 무결성, 순서 보장하지 않음

## Header
- 8 bytes 고정된 길이
- checksum 을 포함하며 IPv6 와 함께 사용 시 checksum 을 필수적으로 포함시켜야함
